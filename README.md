# ntoskrnl7

I build software close to the boundary between systems programming and application runtimes.

My work usually sits around C/C++ runtimes, Windows native APIs, Electron/Chromium internals, IPC, protocol bridges, and TypeScript desktop/web applications. I care about small, explicit interfaces and code that can be moved, tested, and reused without dragging a whole product around it.

## Current Interests

- C/C++ runtime libraries, portability layers, and low-level utilities
- Windows native development, Win32 API wrappers, and desktop integration
- Electron and Chromium internals, including DevTools Protocol and custom protocols
- Type-safe IPC and messaging between native, Node.js, browser, and application layers
- TypeScript application tooling across Electron, Vue, React, and Node.js

## How I Work

I like building the glue that makes layers meet cleanly: runtime code that behaves predictably, native APIs that are easier to use, browser integrations that expose the right amount of power, and messaging contracts that make distributed pieces easier to reason about.

I prefer clear ownership, boring interfaces, and implementation details that are visible when they matter.
